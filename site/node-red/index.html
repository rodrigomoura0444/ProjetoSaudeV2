    Node-RED Flow — ProjetoSaudeV2

    1. Overview

    The Node-RED platform functions as the central logic and automation layer of the system.
    It interconnects the Python simulator, SQLite database, and uibuilder interface — ensuring seamless data flow, validation, and real-time updates across all components.

    2. Main Functions

    • Data Reception: Receives simulated patient data via the /paciente endpoint (HTTP POST).
    • Data Processing and Validation: Parses and verifies the integrity of the incoming JSON payload.
    • Database Insertion: Inserts validated records into the local SQLite database.
    • Dashboard Service: Provides a live visualization interface through uibuilder.
    • API Provision: Exposes the /api/v1/vitals endpoint for historical data retrieval.

    3. Core Nodes Utilized

    Node Type Description
    HTTP In Handles incoming POST requests at /paciente.
    Function Formats, validates, and transforms incoming data.
    SQLite Executes SQL commands to insert or query records.
    HTTP Response Returns acknowledgment responses (HTTP 200 OK).
    uibuilder Manages the frontend interface and real-time data updates.
    HTTP In (GET /api/v1/vitals) Provides access to historical data through an API.

    4. Data Flow

    [HTTP In POST /paciente]
     ↓
[Function: Format external batch]
     ↓
[Function: Build SQL INSERT]
     ↓
[SQLite: INSERT vitals]
     ↓
[HTTP Response 200 OK]

[HTTP In GET /api/v1/vitals]
     ↓
[Function: Build SQL SELECT]
     ↓
[SQLite: SELECT vitals]
     ↓
[Function: Format JSON output]
     ↓
[HTTP Response 200 OK]

    5. API Endpoints

    POST /paciente

    Purpose: Receives vital sign data transmitted by the Python simulator.
    Response: Returns HTTP 200 OK upon successful insertion into the database.

    GET /api/v1/vitals

    Purpose: Allows retrieval of historical patient data.
    Parameters: patient_id, minutes (time window).
    Response: Returns structured JSON data containing vital sign history.

    6. Flow Export

    The complete Node-RED flow configuration is available in the file:
    node-red/flows_MonitoracaoPacientes.json

    This export includes: - Full HTTP input/output configuration.
    - SQLite integration for local data persistence.
    - Real-time dashboard connection via uibuilder.
    - Error handling and data validation logic.

    7. Summary

    The Node-RED flow serves as the core logic engine of ProjetoSaudeV2.
    It connects simulation, data processing, storage, and visualization within a unified, modular, and low-code environment — providing both reliability and educational clarity.
